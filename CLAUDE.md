# CLAUDE.md

이 문서는 Claude Code (claude.ai/code)가 이 저장소에서 작업할 때 따라야 할 가이드를 제공합니다.

## 저장소 성격

"바이브 코딩" 과정 (`doc/바이브_코딩 과정_v3.pdf` 참조)용 실습 저장소입니다. **단일 애플리케이션이 아닙니다** — 여러 참가자가 각자의 실습 결과물을 같은 저장소에 커밋하는 **공유 모노레포**입니다.

## 디렉터리 구조

```
src/exercise/<username>/dayNN/<exercise-name>/
```

- `src/exercise/` 바로 아래의 폴더 하나는 **한 명의 참가자 소유**입니다 (예: `jeonghana0104/`, `0ho0/`, `hyunhong93/`). 참가자는 본인 폴더 내부만 수정합니다.
- 각 실습은 독립적입니다. 보통 단일 파이썬 스크립트(`fibonacci.py`, `pi.py`, `todo.py`)나 작은 정적 웹 앱(HTML/CSS/JS — 마크다운 에디터, 자기소개 랜딩 페이지 등)으로 구성됩니다.
- 공용 모듈도, 폴더 간 import도, 공통 빌드 시스템도 없습니다. **각 실습을 독립된 미니 프로젝트로 취급**하세요.

작업 요청을 받았을 때는 **어떤 참가자의 폴더가 대상인지 먼저 확인**해야 합니다. 같은 주제의 실습이라도 폴더마다 다른 사람의 풀이라서, 폴더를 가로질러 병합하거나 리팩터링하면 안 됩니다.

## 로컬 실행

저장소 어디에도 `package.json`, `requirements.txt`, 린터 설정, 테스트 프레임워크가 **없습니다**. 실습은 다음과 같이 직접 실행합니다.

- **파이썬 실습**: `python3 <경로>/<파일>.py`
- **외부 자산을 분리한 정적 웹 실습** (별도 `.css`/`.js` 파일을 상대 경로로 참조하는 경우): `file://`로 직접 열면 자산 로드가 불안정합니다. 해당 실습 디렉터리에서 간단한 서버를 띄우세요.
  ```
  python3 -m http.server 8765
  ```
  그 다음 브라우저에서 `http://localhost:8765/<파일>.html` 접속. WSL 환경이라면 Windows 쪽 브라우저에서도 같은 URL이 그대로 동작합니다.
- **단일 HTML 파일** 실습: 브라우저에서 그냥 열어도 됩니다.

## Git 워크플로

공유 원격은 `github.com:weable-kosa/kosa-vibecoding-2026-2nd`.

### 정책: 항상 merge, rebase 금지

이 저장소는 **반드시 merge 커밋으로만 통합**해야 합니다. **rebase는 금지**입니다. 이것은 기본값이 아니라 **하드 룰**이며, 모든 pull, 모든 동기화, 모든 "내 브랜치를 최신화" 작업에 적용됩니다.

- pull은 항상 merge로:
  ```
  git pull --no-rebase origin main
  ```
  `git pull --rebase`, `git pull -r`은 **절대 사용하지 않습니다**.
- 공유 브랜치(`main` 등)에 대해 `git rebase`, `git pull --rebase`, `git rebase --onto`를 **실행하지 마세요**.
- "히스토리를 깔끔하게 하려고 main 위로 rebase 할까요?" 같은 **제안도 하지 마세요**. 선형 히스토리는 이 프로젝트가 원하는 모양이 아닙니다.
- `git pull --no-rebase` 도중 충돌이 발생하면 **merge 커밋 안에서 충돌을 해결**합니다. 중단하고 rebase로 갈아타지 마세요.
- 로컬 git 설정이 rebase pull로 되어 있어도 (`pull.rebase=true`), **매번 명령줄에서 `--no-rebase`로 명시적으로 덮어쓰세요**. 사용자 동의 없이 git config를 변경하지 마세요.

**왜인가**: 여러 참가자가 동시에 `main`에 커밋하기 때문에, 팀은 merge 커밋을 통합 기록으로 사용합니다. rebase는 다른 사람이 이미 pull 받았을 수도 있는 커밋 해시를 다시 쓰게 되고, 그러면 그 사람들의 작업본이 깨집니다.

### 그 외 git 습관

- pull 도중 **관련 없는 다른 참가자 폴더의 머지**가 자주 발생합니다. **이게 정상입니다** — 다른 사람 파일을 정리하려 들지 마세요.
- 스테이징할 때는 `git add -A` 대신 **명시적 경로**(`git add src/exercise/<username>/...`)를 사용하세요. 이웃 폴더의 untracked 파일이 작업 트리에 섞여 들어와 실수로 같이 커밋되는 사고를 막을 수 있습니다.
