## 1. 마크다운 에디터

**경로:** `day01/markdown-editor/`

### 구현 내용

순수 HTML/CSS/JS로 실시간 마크다운 에디터를 구현했다. 외부 라이브러리 없이 직접 파서를 작성했다.

### 파일 구조

```
markdown-editor/
├── index.html   # 레이아웃 & 툴바 마크업
├── style.css    # 다크 테마 (CSS 변수 기반)
└── editor.js    # 마크다운 파서 + 에디터 로직
```

### 주요 기능

| 기능 | 설명 |
|------|------|
| 실시간 미리보기 | 입력과 동시에 오른쪽 패널에 렌더링 |
| 마크다운 파서 | 제목/굵기/기울임/취소선/목록/인용/코드블록/링크/표/HR 지원 |
| 툴바 버튼 | 원클릭 서식 삽입 및 토글 |
| 키보드 단축키 | `Ctrl+B` 굵기, `Ctrl+I` 기울임, `Ctrl+K` 링크, `Tab` 들여쓰기 |
| 동기 스크롤 | 에디터↔미리보기 스크롤 연동 (토글 가능) |
| 뷰 모드 전환 | 분할 / 에디터만 / 미리보기만 |
| 드래그 리사이저 | 가운데 구분선 드래그로 패널 비율 조절 |
| 자동 저장 | `localStorage`에 내용 자동 저장 |
| 파일 다운로드 | `.md` 파일로 내보내기 |

### 파서 처리 순서

```
HTML 이스케이프 → 펜스드 코드블록 → 블록 요소(제목·인용·HR·목록) → 인라인 요소(굵기·기울임·링크) → 단락 래핑
```

> 처리 순서를 바꾸면 중첩 패턴이 오파싱되므로 순서 유지가 핵심이다.

### 툴바 액션 3종 구조

```javascript
const SNIPPETS = {
  bold:      { wrap: "**" },       // 선택 텍스트를 감싸는 인라인 서식
  h1:        { prefix: "# " },     // 현재 줄 앞에 접두사 추가 (재클릭 시 토글)
  codeblock: { block: "```\n...\n```" },  // 고정 문자열 삽입
};
```

### 실행

```bash
# 브라우저에서 직접 열기
open day01/markdown-editor/index.html

# 또는 로컬 서버
python3 -m http.server 8080 --directory day01/markdown-editor
```

## 2. 메모 앱 (FastAPI + SQLite)

**경로:** `day01/memo-app/`

### 구현 단계

**1단계** — FastAPI CRUD API + SQLite + 프론트엔드 UI  
**2단계** — JWT 사용자 인증 추가  
**3단계** — 태그/카테고리 기능 추가

### 파일 구조

```
memo-app/
├── main.py          # FastAPI 라우트
├── database.py      # SQLAlchemy + SQLite 연결
├── models.py        # User / Tag / Memo ORM 모델
├── schemas.py       # Pydantic 요청·응답 스키마
├── auth.py          # JWT 발급·검증, 비밀번호 해시
├── requirements.txt
├── .gitignore       # memos.db 제외
└── static/
    ├── index.html   # 로그인 화면 + 메모 앱 레이아웃
    ├── style.css    # 다크 테마
    └── app.js       # API 호출 + UI 로직
```

### DB 스키마

```
users ──< memos
      ──< tags ──<(memo_tags)>── memos  (M:N)
```

### API 엔드포인트

#### 인증
| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/auth/register` | 회원가입 → JWT 반환 |
| POST | `/auth/login` | 로그인 → JWT 반환 |
| GET  | `/auth/me` | 내 정보 (보호) |

#### 태그
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET    | `/tags` | 내 태그 목록 |
| POST   | `/tags` | 태그 생성 |
| DELETE | `/tags/{id}` | 태그 삭제 |

#### 메모
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET    | `/memos` | 전체 목록 (최신순) |
| POST   | `/memos` | 메모 생성 (태그 포함) |
| GET    | `/memos/{id}` | 단일 조회 |
| PUT    | `/memos/{id}` | 수정 |
| DELETE | `/memos/{id}` | 삭제 |

### 프론트엔드 핵심 설계

- **인증 상태**: 페이지 로드 시 `localStorage`의 토큰으로 `/auth/me` 호출 → 성공하면 앱 화면, 실패하면 로그인 화면
- **태그 필터**: 사이드바 태그 칩 클릭 → 클라이언트 사이드 필터링 (추가 API 호출 없음)
- **태그 드롭다운**: 에디터 상단 태그 바에서 미배정 태그만 표시, 클릭으로 추가·제거
- **Ctrl+S**: 어느 화면에서든 저장 단축키 동작

### 실행

```bash
cd day01/memo-app
pip install -r requirements.txt
uvicorn main:app --reload
# → http://localhost:8000
```

## 3. JWT 인증 서비스

**경로:** `day01/jwt-auth/`

### 구현 목표

Access token + Refresh token을 분리하고, 토큰 로테이션까지 포함한 완성형 JWT 인증 서버.

### 파일 구조

```
jwt-auth/
├── main.py       # FastAPI 라우트
├── auth.py       # JWT 발급·검증 로직
├── database.py   # SQLite 연결
├── models.py     # User / RefreshToken 모델
├── schemas.py    # Pydantic 스키마
├── requirements.txt
└── .gitignore
```

### 토큰 전략

| 구분 | 만료 | 저장 위치 | 용도 |
|------|------|-----------|------|
| Access Token | **15분** | 클라이언트 메모리 | API 인증 헤더 |
| Refresh Token | **7일** | DB (`refresh_tokens`) | Access Token 갱신 |

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/auth/register` | 회원가입 → TokenPair |
| POST | `/auth/login` | 로그인 → TokenPair |
| POST | `/auth/refresh` | 토큰 갱신 + 로테이션 → TokenPair |
| POST | `/auth/logout` | refresh token 폐기 |
| GET  | `/auth/me` | 내 정보 (보호) |
| GET  | `/protected/data` | 보호 리소스 예시 |
| GET  | `/protected/tokens` | 활성 세션 목록 (보호) |

### 토큰 로테이션 흐름

```
클라이언트                         서버
    │                               │
    │── POST /auth/refresh ────────>│
    │   { refresh_token: OLD }      │
    │                               ├─ OLD 토큰 revoked = True
    │                               ├─ 새 access_token 발급
    │                               ├─ 새 refresh_token 발급 + DB 저장
    │<─ { access_token: NEW_A  ────│
    │     refresh_token: NEW_R } }  │
    │                               │
    │── POST /auth/refresh ────────>│  (OLD 재사용 시도)
    │   { refresh_token: OLD }      ├─ revoked = True → 401 반환
    │<─ 401 Unauthorized ──────────│
```

### 주요 구현 포인트

**`jti` 클레임으로 토큰 고유성 보장**
```python
# 같은 초에 여러 번 로그인해도 토큰이 달라야 함
payload = {
    "sub": str(user_id),
    "type": "refresh",
    "exp": expire,
    "jti": secrets.token_hex(16),   # 16바이트 랜덤 ID
}
```

**SQLite naive UTC 일관성**
```python
# SQLite는 timezone-aware datetime을 저장하지 못함
# → 저장: datetime.utcnow() (naive)
# → 비교: datetime.utcnow() (naive)
# JWT exp는 jose 라이브러리가 내부 처리

def create_refresh_token(user_id: int) -> tuple[str, datetime]:
    expire = datetime.now(timezone.utc) + timedelta(days=7)
    ...
    return token, expire.replace(tzinfo=None)  # DB 저장용 naive UTC
```

**보호 엔드포인트 의존성 체인**
```python
def get_current_user(
    user_id: int = Depends(auth.decode_access_token),  # JWT 검증
    db: Session = Depends(get_db),
) -> models.User:
    user = db.get(models.User, user_id)
    if not user:
        raise HTTPException(status_code=401)
    return user

@app.get("/protected/data")
def protected(user: models.User = Depends(get_current_user)):
    ...
```

### 테스트 시나리오 (전체 통과)

```
1. 회원가입 OK  (access_expires_in=900s)
2. 로그인 OK
3. 잘못된 비밀번호 → 401
4. /auth/me OK
5. /protected/data OK
6. 토큰 없이 접근 → 401
7. /auth/refresh OK  (토큰 로테이션)
8. 폐기된 refresh token 재사용 → 401
9. 새 access token으로 보호 리소스 접근 → 200
10. /protected/tokens OK
11. /auth/logout → 204
12. 로그아웃 후 refresh → 401
```

### 실행

```bash
cd day01/jwt-auth
pip install -r requirements.txt
uvicorn main:app --reload
# Swagger UI: http://localhost:8000/docs
```
