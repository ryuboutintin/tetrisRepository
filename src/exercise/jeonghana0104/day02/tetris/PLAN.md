# Tetris MVP 계획

## 목표

브라우저에서 바로 실행 가능한 간단한 테트리스 웹 앱. 고급 기능은 빼고 **MVP 수준의 핵심 기능**만. 스팀의 테트리스 비주얼(다크 배경 + 비비드 컬러 블록)을 참고. 프론트엔드 파일은 **타입별로 분리**해 유지보수가 쉽게.

## MVP 기능 범위

**포함**
- 표준 7종 테트로미노 (I, O, T, S, Z, L, J)
- **보너스 블록 3종** (PLUS ✚ / U ∪ / BAR3 ━) — 약 20% 확률로 등장
- 좌/우 이동, 회전, 소프트 드롭, 하드 드롭
- 일시정지 / 재개
- **홀드 (C키)** — 1개만 보관 · 블록 스폰 1회당 1번 사용
- 라인 클리어 (1/2/3/4줄 → 100/300/500/800 × 레벨)
- 점수 / 라인 수 / 레벨 표시
- HOLD · NEXT 미리보기
- 레벨업 시 낙하 속도 가속
- READY / PAUSED / GAME OVER 오버레이
- **랜딩 페이지** — 첫 진입 시 게임 설명 + PLAY 버튼
- **BGM (Korobeiniki)** — Web Audio API로 합성, M키로 토글

**제외 (고급 기능이라 MVP에서 제외)**
- 고스트 피스(낙하 위치 미리보기)
- 효과음 (라인 클리어/하드 드롭 SFX)
- SRS wall kick 등 정교한 회전 보정
- 로컬 스토리지 최고 점수 저장
- 모바일 터치 컨트롤
- 백엔드 (정적 클라이언트만으로 충분)

## 페이지 흐름

```
index.html (랜딩)  ──▶  game.html (게임)
   ▲                      │
   └── ← Back to menu ────┘
```

- `index.html` (랜딩): 첫 진입 페이지. HOW TO PLAY / CONTROLS / SCORING / PIECES 4개 카드로 게임 정보를 보여주고, 큰 ▶ PLAY 버튼으로 게임 페이지 이동.
- `game.html` (게임): 실제 플레이 페이지. 좌측 HOLD + 컨트롤 안내, 중앙 보드, 우측 NEXT + 점수 + START/BGM 버튼.

## 파일 구조 (타입별 분리)

```
tetris/
├── index.html              # 랜딩 페이지
├── game.html               # 게임 페이지
├── PLAN.md                 # 이 문서
├── css/
│   ├── landing.css         # 랜딩 페이지 전용 스타일
│   └── style.css           # 게임 페이지 스타일 (스팀 테트리스 풍 다크 테마)
└── js/
    ├── tetrominoes.js      # 블록 모양/색상 + 표준/보너스 풀 + 랜덤 선택
    ├── board.js            # 보드 상태/렌더링/충돌/라인 클리어
    ├── audio.js            # Web Audio API BGM 플레이어 (Korobeiniki)
    ├── game.js             # TetrisGame 클래스 (게임 로직/점수/레벨/홀드)
    └── main.js             # 진입점, DOM/캔버스/키 입력/오디오 토글 결합
```

각 JS 파일은 책임이 한 가지로 모이게 분리 — 데이터(`tetrominoes`) → 렌더/충돌(`board`) → 오디오(`audio`) → 게임 상태(`game`) → I/O 결합(`main`). 모듈 시스템(import/export) 없이 전역 스크립트 5개로 구성해 빌드 도구나 서버 설정 없이 그대로 돌아가게 함.

## 디자인 방향 (스팀 테트리스 참고)

- 다크 배경 + 라디얼 그라데이션으로 깊이감
- 타이틀에 네온 그라데이션(파랑 → 보라 → 핑크) 텍스트
- 블록은 채도 높은 단색 + 상단/좌측 하이라이트, 하단/우측 섀도우로 입체감
- 게임 사이드 패널: 좌측 HOLD + 컨트롤 / 우측 NEXT + 점수/라인/레벨 + START + BGM 토글

## BGM (Web Audio API)

- 곡: **Korobeiniki** (러시아 민요, public domain — 흔히 Tetris 테마/“Bradinsky”로 불리는 그 멜로디)
- 합성 방식: 오디오 파일 없이 `OscillatorNode`로 실시간 생성
  - 멜로디: square wave (8비트 느낌)
  - 베이스: triangle wave (i-iv-V 진행, A minor)
- 스케줄링: look-ahead 패턴 (`scheduleAhead = 0.15s`, `setTimeout` 25ms 주기로 재시도) — `setTimeout` 지터에 따른 드리프트 방지
- 자동 재생 정책 대응: 사용자가 **START 버튼을 클릭한 시점**에 `AudioContext`를 생성·재생 시작 (사용자 제스처 안에서)
- 음소거: 마스터 게인을 0으로 부드럽게 램프 (`setTargetAtTime`)

## 홀드 (C키)

- 첫 사용 시: 현재 블록을 홀드 슬롯에 보관 → 다음 블록 스폰
- 이미 보관 중일 때: 현재 블록과 홀드 블록 교환 (보드 상단 중앙에 재배치)
- **블록이 스폰될 때마다 `holdLocked = false`**, 한 번 사용하면 `true` → 다음 블록까지 재사용 불가 (홀드 무한 루프 방지)
- 보관된 블록은 회전 상태를 유지하지 않고 **원본 모양**으로 복원해서 꺼낸다

## 블록 종류

| Key | 형태 | 색상 | 비고 |
|---|---|---|---|
| I | 4칸 가로 막대 | #3ae0ee (시안) | 표준 |
| O | 2×2 사각형 | #f5d23a (노랑) | 표준 |
| T | T 모양 | #b870f5 (보라) | 표준 |
| S | S 모양 | #5ce26b (녹색) | 표준 |
| Z | Z 모양 | #ef5b6a (빨강) | 표준 |
| L | L 모양 | #f59a3a (주황) | 표준 |
| J | J 모양 | #5b8bef (파랑) | 표준 |
| PLUS | + 십자 | #ff6dde (핑크) | 보너스 |
| U | U 컵 | #ffd166 (앰버) | 보너스 |
| BAR3 | 3칸 짧은 막대 | #7fffd4 (민트) | 보너스 |

모든 블록은 정사각 행렬로 정의 → `rotateMatrix(matrix)` 한 함수로 회전 처리. 보너스 블록 등장 확률은 `BONUS_RATE = 0.2`.

## 컨트롤

| 키 | 동작 |
|---|---|
| ← / → | 좌우 이동 |
| ↑ | 회전 |
| ↓ | 소프트 드롭 (+1점/칸) |
| Space | 하드 드롭 (+2점/칸) |
| C | 홀드 / 교체 (1회/스폰) |
| P | 일시정지 / 재개 |
| M | BGM ON/OFF |

## 점수 계산

| 동작 | 점수 |
|---|---|
| Single (1줄) | 100 × 레벨 |
| Double (2줄) | 300 × 레벨 |
| Triple (3줄) | 500 × 레벨 |
| Tetris (4줄) | 800 × 레벨 |
| Soft Drop | +1 / 칸 |
| Hard Drop | +2 / 칸 |

라인 10개마다 레벨 +1, 낙하 간격은 `max(100ms, 800 - (level-1)×70)`.

## 실행 방법

외부 CSS/JS를 상대 경로로 참조하므로 `file://`이 아닌 **정적 서버**에서 실행.

```bash
cd src/exercise/jeonghana0104/day02/tetris
python3 -m http.server 8765
```

브라우저에서 `http://localhost:8765/` 접속 → 랜딩 페이지 → ▶ PLAY 클릭 → 게임 페이지에서 **START** 버튼 클릭 (이 시점에 BGM 자동 시작).
