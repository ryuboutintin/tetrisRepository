# Mini Tetris 프로젝트 문서

## 1. 프로젝트 목적

이 프로젝트는 브라우저에서 바로 실행할 수 있는 간단한 테트리스 게임입니다.

목표는 고급 기능이 많은 완성형 게임이 아니라, HTML, CSS, JavaScript만으로 동작하는 MVP 버전의 테트리스 애플리케이션을 만드는 것입니다.

현재 구현된 핵심 목표는 다음과 같습니다.

- 별도 설치 없이 브라우저에서 바로 실행
- 랜딩 페이지에서 게임 방법과 점수 계산 안내
- 플레이 버튼을 누르면 게임 화면으로 이동
- 테트리스 기본 조작 제공
- 점수와 제거한 라인 수 표시
- 다음 블록 미리보기
- Web Audio API를 사용한 배경 음악 재생

## 2. 전체 프로젝트 구조

현재 프로젝트는 순수 프론트엔드 정적 웹앱입니다.

```text
tetris/
├── index.html
├── styles.css
├── script.js
└── README.md
```

현재는 FastAPI 서버, SQLite 데이터베이스, JWT 인증, API 서버 파일이 포함되어 있지 않습니다.

## 3. 각 파일 역할 설명

### `index.html`

웹페이지의 HTML 구조를 담당합니다.

주요 역할은 다음과 같습니다.

- 랜딩 페이지 화면 구성
- 플레이 방법 안내 영역 구성
- 점수 계산 안내 영역 구성
- 게임 페이지 화면 구성
- 테트리스 보드용 `canvas` 배치
- 다음 블록 미리보기용 `canvas` 배치
- 점수, 라인 수, 음악 버튼, 일시정지 버튼, 새 게임 버튼 배치
- `styles.css`와 `script.js` 연결

### `styles.css`

화면 디자인과 반응형 레이아웃을 담당합니다.

주요 역할은 다음과 같습니다.

- 전체 배경색, 텍스트 색상, 버튼 색상 정의
- 랜딩 페이지 디자인
- 게임 화면 카드 레이아웃
- 테트리스 보드와 사이드 패널 배치
- 모바일 화면 대응
- 버튼 hover 효과
- 게임오버 상태 시 보드 테두리 강조

### `script.js`

테트리스 게임 로직과 음악 재생 로직을 담당합니다.

주요 역할은 다음과 같습니다.

- 10 x 20 테트리스 보드 생성
- 7가지 블록 정의: `I`, `J`, `L`, `O`, `S`, `T`, `Z`
- 랜덤 블록 생성
- 블록 좌우 이동
- 블록 아래 이동
- 블록 회전
- 즉시 드롭
- 충돌 검사
- 블록 고정
- 완성된 줄 삭제
- 점수 계산
- 다음 블록 표시
- 게임오버 처리
- 일시정지 처리
- Web Audio API를 사용한 BGM 재생
- 키보드와 버튼 이벤트 연결

### `README.md`

프로젝트 사용법, 구조, 현재 기능, 미구현 기능, 이후 TODO를 설명하는 문서입니다.

## 4. 실행 방법

현재 프로젝트는 서버 없이 실행할 수 있습니다.

### 가장 간단한 실행 방법

1. 프로젝트 폴더를 엽니다.
2. `index.html` 파일을 브라우저로 엽니다.
3. 랜딩 페이지에서 `플레이` 버튼을 누릅니다.

### 로컬 정적 서버로 실행하는 방법

브라우저에서 파일을 바로 열어도 동작하지만, 개발 중에는 정적 서버를 사용하는 것이 편합니다.

Python이 설치되어 있다면 다음 명령어를 사용할 수 있습니다.

```bash
python3 -m http.server 8000
```

그 다음 브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:8000
```

## 6. SQLite DB 구조 설명

현재 프로젝트에는 SQLite DB가 없습니다.

현재 점수와 게임 상태는 브라우저 메모리 안에서만 관리됩니다. 페이지를 새로고침하면 점수와 게임 진행 상태는 초기화됩니다.

향후 점수 저장 기능을 추가한다면 예를 들어 다음과 같은 SQLite 테이블을 만들 수 있습니다.

### `users` 테이블 예시

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  created_at TEXT NOT NULL
);
```

### `scores` 테이블 예시

```sql
CREATE TABLE scores (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER,
  score INTEGER NOT NULL,
  lines INTEGER NOT NULL,
  created_at TEXT NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

현재 구현에서는 위 테이블이 실제로 생성되거나 사용되지 않습니다.

## 7. JWT 인증 흐름 설명

현재 프로젝트에는 JWT 인증이 없습니다.

현재 사용자는 로그인 없이 바로 게임을 플레이합니다. 따라서 토큰 발급, 토큰 저장, 인증 헤더 전송 같은 흐름은 없습니다.

향후 JWT 인증을 추가한다면 일반적인 흐름은 다음과 같습니다.

1. 사용자가 회원가입 또는 로그인을 합니다.
2. FastAPI 서버가 아이디와 비밀번호를 검증합니다.
3. 검증에 성공하면 서버가 JWT access token을 발급합니다.
4. 프론트엔드는 토큰을 저장합니다.
5. 점수 저장 같은 보호된 API 요청을 보낼 때 `Authorization` 헤더에 토큰을 넣습니다.
6. 서버는 토큰을 검증한 뒤 요청을 처리합니다.

예시 헤더는 다음과 같습니다.

```http
Authorization: Bearer <access_token>
```

현재 프로젝트에서는 이 흐름이 아직 구현되어 있지 않습니다.

## 8. CRUD 기능 설명

현재 프로젝트에는 백엔드 CRUD 기능이 없습니다.

CRUD는 보통 다음 네 가지 작업을 의미합니다.

- Create: 데이터 생성
- Read: 데이터 조회
- Update: 데이터 수정
- Delete: 데이터 삭제

현재 게임은 브라우저 안에서만 동작하므로 서버에 데이터를 생성하거나 조회하지 않습니다.

향후 점수판 기능을 추가한다면 다음과 같은 CRUD가 가능해질 수 있습니다.

- 점수 생성: 게임 종료 후 점수 저장
- 점수 조회: 랭킹 목록 가져오기
- 점수 수정: 관리자 기능으로 점수 수정
- 점수 삭제: 관리자 기능으로 점수 삭제

## 9. 기능 설명

### 랜딩 페이지

처음 접속하면 바로 게임이 시작되지 않고 랜딩 페이지가 표시됩니다.

랜딩 페이지에는 다음 정보가 표시됩니다.

- 프로젝트 이름
- 간단한 게임 소개
- 플레이 방법
- 점수 계산 방식
- 플레이 버튼

### 게임 시작

랜딩 페이지에서 `플레이` 버튼을 누르면 게임 페이지가 표시되고 새 게임이 시작됩니다.

브라우저 보안 정책상 음악은 자동으로 재생할 수 없습니다. 그래서 사용자가 `플레이` 버튼을 클릭한 뒤 Web Audio API 음악이 시작됩니다.

### 블록 이동

현재 블록은 키보드 또는 화면 버튼으로 조작할 수 있습니다.

- 왼쪽 이동
- 오른쪽 이동
- 아래 이동
- 회전
- 즉시 드롭

### 점수 계산

현재 점수 계산은 다음 규칙을 사용합니다.

| 동작 | 점수 |
| --- | ---: |
| 1줄 제거 | 100 |
| 2줄 제거 | 300 |
| 3줄 제거 | 500 |
| 4줄 제거 | 800 |
| 즉시 드롭 | 내려간 칸마다 2 |

### 게임오버

새 블록이 생성되는 위치에 이미 블록이 쌓여 있으면 게임오버가 됩니다.

게임오버가 되면 보드 위에 `GAME OVER` 문구가 표시됩니다.

### 일시정지

`P` 키 또는 상단의 일시정지 버튼을 누르면 게임이 멈춥니다.

다시 누르면 게임이 재개됩니다.

### 배경 음악

`script.js`에서 Web Audio API를 사용해 테트리스풍 BGM을 재생합니다.

별도의 mp3 파일을 사용하지 않고, 브라우저의 오디오 엔진에서 직접 음을 합성합니다.

사용한 주요 Web Audio API 기능은 다음과 같습니다.

- `AudioContext`
- `OscillatorNode`
- `GainNode`
- `BiquadFilterNode`

## 10. API 엔드포인트 정리

현재 프로젝트에는 API 엔드포인트가 없습니다.

현재는 모든 기능이 프론트엔드에서만 실행됩니다.

향후 FastAPI 백엔드를 추가한다면 예를 들어 다음과 같은 API를 만들 수 있습니다.

| Method | Endpoint | 설명 | 현재 상태 |
| --- | --- | --- | --- |
| `POST` | `/auth/register` | 회원가입 | 미구현 |
| `POST` | `/auth/login` | 로그인 및 JWT 발급 | 미구현 |
| `GET` | `/scores` | 점수 랭킹 조회 | 미구현 |
| `POST` | `/scores` | 점수 저장 | 미구현 |
| `GET` | `/scores/me` | 내 점수 조회 | 미구현 |
| `DELETE` | `/scores/{score_id}` | 점수 삭제 | 미구현 |

## 14. git add / commit / pull / push 흐름 설명

Git은 코드 변경 이력을 관리하는 도구입니다.

일반적인 작업 흐름은 다음과 같습니다.

### 1단계: 현재 변경 상태 확인

```bash
git status --short
```

어떤 파일이 새로 생겼고 어떤 파일이 수정되었는지 확인합니다.

### 2단계: 변경 내용 확인

```bash
git diff
```

내가 어떤 코드를 바꿨는지 확인합니다.

### 3단계: 커밋할 파일 추가

```bash
git add index.html styles.css script.js README.md
```

또는 현재 폴더의 모든 변경 파일을 추가하려면 다음 명령어를 사용할 수 있습니다.

```bash
git add .
```

### 4단계: 커밋 생성

```bash
git commit -m "Add mini tetris app documentation"
```

커밋은 변경 사항을 하나의 기록으로 저장하는 작업입니다.

### 5단계: 원격 저장소 변경 내용 가져오기

```bash
git pull
```

다른 사람이 원격 저장소에 올린 변경 사항을 내 로컬 저장소로 가져옵니다.

### 6단계: 원격 저장소에 올리기

```bash
git push
```

내 커밋을 원격 저장소에 업로드합니다.

## 18. 초보자를 위한 개발 흐름 요약

이 프로젝트를 처음 보는 경우 다음 순서로 이해하면 좋습니다.

1. `index.html`을 열어 화면 구조를 확인합니다.
2. `styles.css`를 열어 화면이 어떻게 꾸며졌는지 확인합니다.
3. `script.js`를 열어 게임 규칙과 조작 로직을 확인합니다.
4. 브라우저에서 `index.html`을 실행합니다.
5. 랜딩 페이지에서 `플레이` 버튼을 누릅니다.
6. 키보드 방향키와 스페이스바로 게임을 조작합니다.
7. 코드를 수정한 뒤 브라우저를 새로고침해서 결과를 확인합니다.

현재 프로젝트는 서버 없이 동작하기 때문에 백엔드 지식이 없어도 실행과 수정이 쉽습니다.
