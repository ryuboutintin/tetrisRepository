# Workflow

이 문서는 이 프로젝트를 다룰 때 유지할 작업 기준을 정리합니다.

## 기본 원칙

- 구현 전에 먼저 방법을 설명한다.
- 변경 사항은 코드만이 아니라 문서로도 남긴다.
- 사용자가 다음에 다시 말하지 않아도 되도록, 프로젝트 안에 기준을 보관한다.

## 권장 진행 순서

1. 현재 파일 구조와 요구 사항을 확인한다.
2. 어떤 방식으로 바꿀지 먼저 공유한다.
3. 코드와 문서를 함께 수정한다.
4. 가능하면 바로 실행 또는 문법 검증을 한다.

## 문서 업데이트 대상

- `README.md`
- 필요한 경우 기능별 보조 문서
- 사용자가 특별히 요청한 작업 기준 문서
